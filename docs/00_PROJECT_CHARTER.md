# Project Charter

## 문서 상태

- 단계: Phase 0 기획
- 성격: 사용자와 합의한 목표를 정리한 기준 문서
- 주의: 아래 내용은 구현 완료를 뜻하지 않는다.

## 한 문장 정의

CT Cine Reviewer(CCR)는 전달받은 CT cine 동영상을 Windows에서 프레임 단위로 검토하고, 화면 픽셀 기반 보정과 주석을 적용해 이미지를 공유할 수 있게 하는 완전 로컬 보조 뷰어이다.

## 문제 정의

일반 동영상 플레이어는 시간 기반 재생에 최적화되어 있어 CT cine 영상을 슬라이스처럼 한 프레임씩 앞뒤로 확인하기 어렵다. 확대·이동 상태를 유지하면서 탐색하거나, 의심 부위를 표시하고, 보정과 주석이 포함된 한 장면을 공유하는 흐름도 불편하다.

## 주 사용자

- 주 사용자: 카카오톡 등으로 CT cine 동영상을 전달받아 빠르게 검토하는 의료인
- 보조 사용자: 사용 환경을 설치·점검하고 공유 이미지를 준비하는 사용자
- 전제: v0.1의 현실적인 입력은 MP4이며, DICOM 원본 접근은 보장되지 않는다.

## 대표 사용 시나리오

1. 사용자가 전달받은 MP4를 앱에 끌어 놓는다.
2. 앱이 첫 프레임과 영상 정보를 보여준다.
3. 사용자가 휠이나 방향키로 정확히 한 프레임씩 탐색한다.
4. 확대·Pan과 Video Display Preset 또는 수동 보정을 사용한다.
5. 특정 프레임에 화살표와 텍스트를 추가하고 개인정보 영역을 가린다.
6. 원본 해상도 기준으로 주석 포함 PNG를 저장한다.
7. 필요하면 별도 프로젝트 파일로 작업 상태를 보존한다.

## 해결하려는 불편

- 시간 seek 때문에 생기는 불정확한 한 프레임 이동
- 이전 프레임 이동의 지연과 불안정성
- 확대·Pan 상태에서의 탐색 불편
- 동영상 프레임에 수정 가능한 주석을 남기기 어려움
- 보정·주석·마스킹을 포함한 공유 이미지 생성의 번거로움

## 프로젝트 목표

- 프레임 인덱스와 PTS를 구분해 안정적인 전후 탐색을 제공한다.
- MP4의 한계를 숨기지 않으면서 화면 픽셀 기반 표시 보정을 제공한다.
- 원본 영상은 변경하지 않고 주석과 상태를 별도 저장한다.
- 환자 자료가 외부로 전송되지 않는 로컬 전용 흐름을 유지한다.
- 실제 대표 영상으로 각 단계의 정확도와 성능을 검증한다.

## 비목표

- 의료기기 인증을 받은 진단 시스템
- PACS 또는 원본 DICOM 판독의 대체
- MP4에서 HU 복원 또는 실제 DICOM window 제공
- v0.1의 DICOM, PACS, AI 판독, 클라우드, 협업 계정

## 의료적·기술적 한계

- MP4는 대개 PACS에서 이미 표시 변환된 8비트 영상이므로 원본 HU, bit depth, pixel spacing, slice thickness 및 DICOM 메타데이터가 없다.
- 압축·재인코딩 과정에서 사라진 영상 정보는 Sharp나 Denoise로 복원할 수 없다.
- 강한 보정은 잡음, halo 또는 가짜 경계를 만들 수 있으므로 원본 비교가 필요하다.
- 프레임 순서가 영상의 실제 해부학적 slice 순서를 완전히 보장한다는 전제는 입력 영상에 의존한다.
- 공유 이미지의 개인정보 제거 책임을 앱이 완전히 자동 보장할 수는 없다.

## 성공의 정의

v0.1은 다음을 실제 샘플 영상에서 함께 만족할 때 성공으로 본다.

- 전후 한 프레임 탐색에서 누락·중복·순서 역전이 검출되지 않는다.
- 확대·Pan 상태를 유지한 탐색과 주요 단축키가 실사용 가능하다.
- 표시 보정과 주석이 화면과 내보낸 이미지에서 일관된다.
- 원본 파일이 변경되지 않으며 임시 캐시가 정상·비정상 종료 후 정리된다.
- MP4 보정이 HU 기반 window로 오인되지 않는 UI와 문구를 유지한다.
- 대표 샘플과 성능 기준은 기술 스파이크에서 측정해 사용자 승인을 받는다.
