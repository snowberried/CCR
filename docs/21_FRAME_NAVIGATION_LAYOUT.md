# Frame Navigation & Right Panel Tabs

## 결론

v0.5.1 UI 기준선을 보존하면서 하단 프레임 탐색을 시각적으로 정리했다. 최초에는 오른쪽의 조정 기능과 상태 정보를 독립 컬럼으로 분리했으나, v0.5.2 사용자 결정으로 하나의 `조정 / 정보` 탭 패널로 통합했다. 동영상 자동 재생과 미디어 컨트롤은 제품 범위에서 계속 제외한다.

이 앱은 계속 CT cine MP4 검토를 돕는 로컬 보조 뷰어이다. 의료기기·공식 PACS·DICOM 판독 프로그램이 아니다.

## 하단 탐색 UI

- 위 행은 주석 marker, 진행 구간, playhead와 눈금이 보이는 시각적 타임라인이다.
- 아래 행은 현재/전체 시간과 정밀 프레임 이동 컨트롤로 구성한다.
- 현재/전체 시간은 `HH:MM:SS / HH:MM:SS`로 표시하고 24시간을 넘어도 누적 시간을 유지한다.
- 정밀 이동은 첫 프레임, 빠른 이전 N, 이전 1, 직접 프레임 입력, 다음 1, 빠른 다음 N, 마지막 프레임을 제공한다. 빠른 이동 기본값은 5프레임이다.
- 하단은 `왼쪽 설정 | 중앙 정밀 탐색 | 오른쪽 확대율`의 3구역이다. 정밀 탐색은 양쪽 유틸리티 사이 가용 영역의 중앙에 두고, 주석 기본색과 같은 `#FFD54F` 2px·80% opacity 구분선은 각 빈 여백의 중앙에 둔다. 탐색 자체의 3:1:3 폭은 유지한다.
- `설정`은 독립 설정창을 여는 테두리 없는 버튼이며 글자는 13px이다. 설정창에는 빠른 이동 간격, 전용 단축키 설정 진입과 사용자가 직접 실행하는 GitHub 최신 Release 확인을 제공한다.
- 빠른 이동 간격은 2·5·10·20·30·50프레임 목록과 2~999프레임 직접 입력을 제공하고, 로컬 설정에 저장해 다음 실행에도 유지한다.
- 확대율 셀은 항상 원본 픽셀 기준의 현재 실제 배율을 표시한다. 클릭하면 `화면 맞춤, 50%, 75%, 100%, 125%, 150%, 175%, 200%`를 선택할 수 있고 별도 화면 맞춤 버튼은 두지 않는다.
- `−/+`는 현재 실제 배율을 10%p씩 조정한다. 새 영상은 화면 맞춤으로 시작하고, 화면 맞춤 선택 뒤에도 셀에는 계산된 현재 `%`를 표시한다.
- 기본 단축키는 좌우 방향키가 1프레임, 위 방향키가 빠른 다음, 아래 방향키가 빠른 이전이며 Home/End는 첫/마지막 프레임이다. Shift+좌우 보조 조합은 사용하지 않는다. 키보드 이동키는 전용 단축키 설정에서 기능당 하나를 변경하거나 해제할 수 있고 wheel 이동은 고정한다.
- 전용 단축키 창은 변경 초안을 취소할 수 있고, 중복·예약·고정 키 충돌이 없을 때만 한 번에 저장한다. 설정된 빠른 이동키도 같은 N과 반복 이동·방향성 캐시를 사용한다.
- 재생·정지·이전/다음 미디어 버튼과 자동 재생 타이머는 두지 않는다.
- 기존 `취소` 셀은 제거했다. 파일/세션 전환과 오류 복구에 필요한 내부 취소 처리는 유지한다.

## 오른쪽 탭 패널

오른쪽에는 같은 폭을 공유하는 `조정 / 정보` 탭을 둔다. 앱 시작 시 기본 탭은 `조정`이며, 탭을 누르면 해당 내용만 표시한다. 두 내용은 상태 보존을 위해 DOM에 유지하되 비활성 내용은 숨긴다.

`조정` 탭에는 사용자가 값을 바꾸는 항목만 둔다.

- 화면 보정
- 주석 속성·Undo/Redo
- PNG/clipboard 내보내기

`정보` 탭에는 읽기 전용 상태를 둔다.

- 현재/전체 프레임
- 현재 PTS 시간, 영상 길이, 해상도, FPS, 코덱
- cache, 요청 시간, renderer/view/display 진단
- 오류 메시지

1440px에서는 탭 패널 294px을 사용한다. 1180px 이하에서는 218px, 820px 이하에서는 180px로 축소하며 탭과 상단 명령을 모두 유지한다.

## 범위 경계

- 자동 재생과 오디오 재생은 구현하지 않는다.
- `frameIndex`와 실제 PTS를 기존 방식대로 유지하며 FPS로 프레임 위치를 추정하지 않는다.
- decoder/cache hot path와 Phase 5 shared frame/crosshair 의미를 변경하지 않는다.
- MP4 화면 픽셀 보정은 실제 DICOM HU Window가 아니다.
- annotation과 작업 상태는 앱 종료 후 저장되지 않는다.
