# Frame Navigation & Panel Separation

## 결론

v0.5.1 UI 기준선을 보존하면서 하단 프레임 탐색을 시각적으로 정리하고, 오른쪽의 조정 기능과 상태 정보를 독립 컬럼으로 분리했다. 검토 결과 동영상 자동 재생과 미디어 컨트롤은 제품 범위에서 다시 제외했다. 새 배포 버전은 아직 결정하지 않았으며 `package.json`은 0.5.1을 유지한다.

이 앱은 계속 CT cine MP4 검토를 돕는 로컬 보조 뷰어이다. 의료기기·공식 PACS·DICOM 판독 프로그램이 아니다.

## 하단 탐색 UI

- 위 행은 주석 marker, 진행 구간, playhead와 눈금이 보이는 시각적 타임라인이다.
- 아래 행은 현재/전체 시간과 정밀 프레임 이동 컨트롤로 구성한다.
- 현재/전체 시간은 `HH:MM:SS / HH:MM:SS`로 표시하고 24시간을 넘어도 누적 시간을 유지한다.
- 정밀 이동은 첫 프레임, -5, 직접 프레임 입력, +5, 마지막 프레임을 제공한다.
- 키보드 좌우 방향키와 Shift+좌우, Home/End, wheel 기반 기존 프레임 이동은 유지한다.
- 재생·정지·이전/다음 미디어 버튼과 자동 재생 타이머는 두지 않는다.
- 기존 `취소` 셀은 제거했다. 파일/세션 전환과 오류 복구에 필요한 내부 취소 처리는 유지한다.

## 오른쪽 컬럼

`조정` 컬럼에는 사용자가 값을 바꾸는 항목만 둔다.

- 화면 보정
- 주석 속성·Undo/Redo
- PNG/clipboard 내보내기

`정보` 컬럼에는 읽기 전용 상태를 둔다.

- 현재/전체 프레임
- 현재 PTS 시간, 영상 길이, 해상도, FPS, 코덱
- cache, 요청 시간, renderer/view/display 진단
- 오류 메시지

1440px에서는 조정 236px, 정보 210px을 사용한다. 720px에서는 조정 188px, 정보 172px로 축소하며 두 컬럼과 상단 명령을 모두 유지한다. 560px 이하에서는 영상 영역을 보존하기 위해 두 패널을 오른쪽 한 열의 위·아래로 배치한다.

## 범위 경계

- 자동 재생과 오디오 재생은 구현하지 않는다.
- `frameIndex`와 실제 PTS를 기존 방식대로 유지하며 FPS로 프레임 위치를 추정하지 않는다.
- decoder/cache hot path와 Phase 5 shared frame/crosshair 의미를 변경하지 않는다.
- MP4 화면 픽셀 보정은 실제 DICOM HU Window가 아니다.
- annotation과 작업 상태는 앱 종료 후 저장되지 않는다.
