# Phase 2.1 Windows Installer Pilot

## 결론

2026-07-11에 NSIS Setup의 실제 현재 사용자 설치, 설치본 실행, 제거와 깨끗한 재설치를 검증했다. 자동 테스트 34개, 프로덕션 빌드와 패키지 개인정보 검사는 모두 통과했다.

재설치된 `CT Cine Reviewer 0.2.0-phase2`는 현재 설치된 상태다. Phase 3와 제품 기능은 승인되지 않았으므로 시작하지 않는다.

아직 완료로 처리하지 않는 항목은 다음 두 가지다.

- Windows Explorer에서 설치본으로 직접 끌어놓는 실제 drag/drop
- Windows 설정의 설치된 앱 목록과 제거 등록 확인

레지스트리 조회에서는 설치 중 `CT Cine Reviewer` 제거 등록을 찾지 못했다. 설치 폴더에는 공식 제거 프로그램이 정상 생성됐고 제거 자체는 통과했지만, 더 넓은 파일럿 전에는 Windows 설정 UI에서 등록 여부를 다시 확인해야 한다.

## 설치본 기능 회귀

실제 로컬 샘플은 일반 이름 하드링크로만 열었고 원본 파일명과 전체 경로는 기록하지 않았다.

| 샘플 | 표시 결과 |
| --- | --- |
| Sample A | 406×720, 24fps, H.264, 2,560프레임 |
| Sample B | 406×720, 24fps, H.264, 518프레임 |
| Sample C | 406×720, 24fps, H.264, 359프레임 |

Sample A에서 다음 탐색을 확인했다.

- `Right`: 1→2
- `Shift+Right`: 2→7
- `End`: 2,560
- `Home`: 1
- 직접 입력: 100
- 휠 한 동작: 100→101
- 최종 cache hit: 0.1ms
- cache 사용량: 68.0MiB / 72.0MiB

비의료 30프레임 합성 영상에서는 빠른 정방향 50회 입력이 마지막 프레임, 빠른 역방향 50회 입력이 첫 프레임에 정확히 수렴했다. 큰 샘플 분석 직후 취소했을 때 화면이 초기화되고 ffprobe와 FFmpeg 프로세스가 남지 않았다.

## 프로세스·네트워크·원본 보존

- 설치본 유휴 상태에서 앱 프로세스의 network endpoint와 established connection은 0건이었다.
- 앱 종료 후 `CT Cine Reviewer`, ffmpeg, ffprobe 잔존 프로세스는 0건이었다.
- 앱 이름과 일치하는 OS 임시 항목은 0건이었다.
- 제거 전후 Sample A/B/C SHA-256이 모두 일치했다.
- 원본 영상은 수정하거나 재인코딩하지 않았다.

네트워크 검사는 해당 시점의 프로세스 연결 관찰이며 별도 패킷 캡처나 장시간 방화벽 감사 결과는 아니다.

## 제거와 재설치

Windows 자동화 정책이 제거 프로그램 UI 실행을 차단해 공식 NSIS 제거 프로그램을 `/S` 모드로 실행했다.

- 제거 exit code 0
- 설치 폴더 제거
- 바탕화면·시작 메뉴 바로가기 제거
- 확인한 앱 데이터 후보 제거
- 앱·ffmpeg·ffprobe 잔존 프로세스 0
- Sample A/B/C hash 보존
- 같은 Setup으로 깨끗한 재설치 통과
- 재설치 후 실행 파일, 제거 프로그램, 바로가기와 번들 FFmpeg/라이선스 재생성 확인

동일 버전 덮어쓰기 설치는 사용자 결정으로 생략했다. 현재 단계에는 보존해야 할 사용자 프로젝트 데이터가 없어 깨끗한 제거·재설치보다 추가 실익이 작다고 판단했다.

## 남은 파일럿 항목

1. 사용자가 Windows Explorer에서 실제 MP4를 설치본 창으로 drag/drop한다.
2. Windows 설정의 설치된 앱 목록에서 `CT Cine Reviewer`와 제거 동작을 확인한다.
3. 등록이 보이지 않으면 NSIS uninstall registry 생성을 진단하고 수정 범위를 승인받는다.
4. 실제 사용자 파일럿에서 휠 방향, 프레임 감각, 세로 화면 크기와 가장 큰 불편을 기록한다.
5. 실제 사용 피드백과 별도 승인 전에는 Phase 3 기능을 시작하지 않는다.
