# Product Requirements

## 범위와 우선순위

- P0: 없으면 제품의 존재 의미가 없는 기능
- P1: 초기 실사용에 필요한 기능
- P2: 이후 추가 가능한 편의 기능

| 우선순위 | 요구사항 | v0.1 수용 기준 초안 |
| --- | --- | --- |
| P0 | MP4 파일 열기 | 지원 샘플에서 첫 프레임·전체 프레임 수·기본 메타데이터를 표시하고 실패 시 앱이 종료되지 않는다. |
| P0 | 인덱스 기반 전후 한 프레임 탐색 | 휠과 방향키 1회당 인덱스가 정확히 1씩 변하고 첫·끝 범위를 넘지 않는다. |
| P0 | 프레임 인덱스와 PTS 관리 | 가변 프레임레이트 입력에서도 화면 인덱스와 해당 PTS 연결을 유지한다. |
| P0 | 확대·축소·Pan·화면 맞춤 | 프레임 이동 후에도 보기 상태가 유지되고 초기화가 가능하다. |
| P0 | 완전 로컬 처리와 원본 보존 | 네트워크 없이 동작하며 원본 파일의 내용·수정 시각을 바꾸지 않는다. |
| P1 | 재생·정지와 임의 프레임 이동 | 선택한 재생 수준과 속도에서 순서가 유지되고, 번호 입력으로 유효한 프레임에 이동한다. |
| P1 | Video Level/Width, Gamma, Inverse | 변경 즉시 화면에 반영되고 초기화 및 수치 확인이 가능하다. |
| P1 | Sharp와 원본 비교 | 보정 강도를 조절하고 누르는 동안 모든 보정을 해제해 원본을 비교할 수 있다. |
| P1 | Video Display Preset | preset이 픽셀 보정임을 표시하고 HU 단위를 사용하지 않는다. |
| P1 | 화살표·텍스트·기본 도형 | 프레임별로 생성·선택·이동·수정·삭제할 수 있다. |
| P1 | Undo/Redo와 주석 숨김 | 현재 세션의 주석 편집을 되돌리고 원본을 변경하지 않은 채 표시를 숨길 수 있다. |
| P1 | 개인정보 마스킹 | 지정 영역이 내보낸 결과에 적용되며 저장 전 결과를 확인할 수 있다. |
| P1 | 세 종류 이미지 저장 | 원본, 보정만, 보정+주석 결과를 원본 프레임 해상도 기준 PNG로 구분 저장한다. |
| P1 | 프로젝트 저장·불러오기 | 원본 식별자, 프레임/PTS, 보정, 주석, preset, 북마크를 별도 파일로 복원한다. |
| P1 | 임시 캐시 정리 | 정상 종료 시 삭제하고 다음 실행 때 잔여 캐시를 탐지·정리한다. |
| P2 | MOV/AVI/MKV 지원 | 기술 스파이크로 검증한 형식만 지원 목록에 명시한다. |
| P2 | 자유선·북마크·구간 반복 | 핵심 흐름을 방해하지 않으며 저장·복원이 가능하다. |
| P2 | JPEG 내보내기·Crop | 품질 옵션과 결과 영역을 저장 전에 확인할 수 있다. |
| P2 | 최근 파일 목록·전체 화면 | 개인정보 설정에 따라 끌 수 있고 경로·썸네일을 불필요하게 보존하지 않는다. |

## 대표 사용자 흐름

### 정상 흐름

열기 → 영상 분석 → 첫 프레임 표시 → 프레임 탐색 → 표시 보정 → 주석/마스킹 → 저장 미리보기 → PNG 내보내기 → 필요 시 프로젝트 저장

### 프로젝트 재개 흐름

프로젝트 파일 열기 → 원본 식별 확인 → 일치하면 상태 복원 → 이동했으면 원본 재지정 요청 → 변경됐으면 경고 후 사용자 선택 대기

## 오류 상황과 기대 동작

| 상황 | 기대 동작 |
| --- | --- |
| 지원하지 않는 코덱·컨테이너 | 원인을 가능한 범위에서 알려주고 원본과 기존 작업을 보존한다. |
| 손상되거나 불완전한 영상 | 읽은 범위와 실패 지점을 알리고 앱 전체가 종료되지 않는다. |
| 총 프레임 수를 신뢰할 수 없음 | 실제 디코딩 결과를 기준으로 갱신하고 추정값을 확정값처럼 표시하지 않는다. |
| 디스크·메모리 부족 | 캐시 작업을 중단하고 정리 후 사용자가 다른 방식을 선택할 수 있게 한다. |
| 프로젝트 원본이 이동됨 | 자동으로 임의 파일을 연결하지 않고 재지정을 요청한다. |
| 프로젝트 원본이 변경됨 | fingerprint 불일치를 경고하고 주석 오정렬 가능성을 설명한다. |
| 저장 대상이 쓰기 불가 | 원본·프로젝트 상태를 유지하고 다른 경로 선택을 안내한다. |
| 비정상 종료 잔여 캐시 | 다음 시작 때 앱 전용 캐시만 안전하게 정리한다. |

## 성능 목표 초안

아래 수치는 **검증 전 목표**이며 보장이 아니다. 대표 샘플의 해상도·코덱·길이를 정한 후 확정한다.

- 첫 화면 표시: 대표 영상에서 2초 이내 목표
- 캐시된 한 프레임 이동: 입력 후 약 50ms 이내 표시 목표
- 연속 전후 왕복: 누락·중복·순서 역전 0건
- 조작 중 UI: 사용자가 멈춤으로 느끼는 장시간 블로킹 없음
- 내보내기: 화면과 저장 결과의 보정·주석 좌표가 허용 오차 내 일치
